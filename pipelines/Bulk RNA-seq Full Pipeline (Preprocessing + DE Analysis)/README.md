# Bulk RNA-seq Pipeline (Preprocessing + DE Analysis)

A two-step bulk RNA-seq analysis pipeline that performs preprocessing with nf-core/rnaseq (STAR-Salmon) followed by differential expression analysis, enrichment, and GSEA using edgeR and clusterProfiler in R.

## Pipeline Overview

| Step | Name | Tool | Description |
|------|------|------|-------------|
| 1 | Preprocessing | nf-core/rnaseq (STAR-Salmon) | QC, alignment, and transcript quantification |
| 2 | DE Analysis | edgeR + clusterProfiler + fgsea | DEG, enrichment (GO/KEGG/Reactome/Hallmark), GSEA, PCA, volcano plots |

## Required Inputs

### Step 1: Preprocessing
- **FASTQ files**: Paired-end RNA-seq reads (`.fastq.gz`)
- **FASTQ metadata CSV**: Sample sheet with columns `sample`, `fastq_1`, `fastq_2`, `strandedness`
- **Reference genome FASTA**: e.g., `Homo_sapiens.GRCh38.dna_sm.primary_assembly.fa`
- **Reference GTF**: e.g., `Homo_sapiens.GRCh38.112.gtf`

### Step 2: DE Analysis
- **Salmon metadata CSV**: Columns `sample`, `file` (path to `quant.sf`), group column (e.g., `Condition`), `Batch`
- **tx2gene.tsv**: Transcript-to-gene mapping (auto-generated by Step 1)
- **(Optional)** Custom GMT gene set file
- **(Optional)** Hallmark GMT gene set file

## Expected Outputs

### Step 1 (Preprocessing)
- `star_salmon/` — STAR alignment + Salmon quantification per sample
- `star_salmon/tx2gene.tsv` — Transcript-to-gene mapping
- MultiQC report

### Step 2 (DE Analysis)
- `TPM_gene.csv`, `TPM_gene_log2p1.csv` — TPM expression tables
- `TMM_length_normalized_count.csv` — TMM normalized counts
- `PCA_plot.png` — PCA visualization
- `0.DGE/` — DEG result Excel files
- `1.volcano/`, `1.volcano_annot/`, `1.volcano_specific_genes/` — Volcano plots
- `2.DEG_enrichment/` — GO/KEGG/Reactome/Hallmark enrichment (Excel + figures)
- `3.GSEA/` — GSEA results (Excel + bar plots + individual gene set plots)

## How to Run

### 1. Build the Docker image

```bash
docker build -t bulk-rnaseq-pipeline .
```

### 2. Run the pipeline

```bash
docker run --rm \
  -v /path/to/input/data:/input:ro \
  -v /path/to/output:/output \
  -v /var/run/docker.sock:/var/run/docker.sock \
  bulk-rnaseq-pipeline \
  snakemake --cores 8 --snakefile /pipeline/Snakefile
```

**Mount points:**
- `/input` — Directory containing FASTQ files, reference genome, metadata CSVs, and gene set GMT files
- `/output` — Directory where all results will be written

### 3. Run only Step 2 (DE Analysis)

If preprocessing has already been completed:

```bash
docker run --rm \
  -v /path/to/input/data:/input:ro \
  -v /path/to/output:/output \
  bulk-rnaseq-pipeline \
  snakemake --cores 1 --snakefile /pipeline/Snakefile de_analysis
```

## Configuration

All configurable parameters are in `config.yaml`. Key settings:

| Parameter | Description | Default |
|-----------|-------------|---------|
| `preprocessing.aligner` | Alignment method | `star_salmon` |
| `preprocessing.nfcore_revision` | nf-core/rnaseq version | `3.22.2` |
| `de_analysis.group_col` | Metadata column for comparisons | `Condition` |
| `de_analysis.comb_set` | Pairwise comparisons (semicolon-separated) | `ConditionA,ConditionB` |
| `de_analysis.organism` | Species: `human` or `mouse` | `human` |
| `de_analysis.keytype` | Gene ID type: `SYMBOL`, `ENSEMBL`, `ENTREZID` | `SYMBOL` |
| `de_analysis.fc_threshold` | log2FC cutoff | `1` |
| `de_analysis.pval_threshold` | Adjusted p-value cutoff | `0.05` |

## Notes

1. Docker socket must be mounted for Step 1 (nf-core/rnaseq uses Docker internally).
2. Reference genome and GTF must be provided by the user — they are not bundled.
3. The `file` column in Salmon metadata CSV must point to valid `quant.sf` file paths.
4. Supported organisms: `human` (org.Hs.eg.db) and `mouse` (org.Mm.eg.db).
